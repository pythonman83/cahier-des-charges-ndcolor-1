1 Présentation du projet

ND COLOR est une application web moderne de gestion des cartes de fidélité destinée à un salon de coiffure situé à Draguignan.
Cette application permettra de gérer efficacement sa clientèle et son programme de fidélité avec un design premium reflétant l'image haut de gamme de l'établissement.

Informations générales

Nom du projet ND COLOR - Gestion cartes de fidélité
Client Salon de coiffure ND COLOR, Draguignan
Type d'application Application Web Single Page (SPA)
Technologies imposées HTML5, CSS3, Bootstrap 5.3.0, JavaScript Vanilla
Stockage des données localStorage (navigateur)

2 Contexte et enjeux

Le salon ND COLOR souhaite moderniser sa gestion de la fidélité client.
Actuellement, le suivi est effectué manuellement avec des cartes papier, ce qui engendre des pertes d'informations et une expérience client sous-optimale.

Problématiques actuelles

  • Gestion manuelle chronophage des cartes de fidélité papier
  • Perte fréquente d'informations clients
  • Difficulté à identifier les clients les plus fidèles
  • Absence de statistiques sur la clientèle
  • Image de marque non reflétée dans les outils de gestion

Bénéfices attendus

  • Centralisation des données clients dans une interface unique
  • Gain de temps significatif dans la gestion quotidienne
  • Meilleure connaissance de la clientèle via les statistiques
  • Fidélisation accrue grâce au système de points
  • Image de marque premium renforcée
Note pédagogique :
Ce projet vous permet de mettre en pratique les compétences CRUD (Create, Read, Update, Delete) essentielles pour tout développeur web, tout en travaillant sur un design premium et une UX soignée.

3 Objectifs du projet

Objectifs fonctionnels

  • Permettre l'ajout, la modification et la suppression de clients
  • Gérer un système de points de fidélité avec 4 niveaux de cartes
  • Proposer une recherche instantanée dans la liste des clients
  • Afficher des statistiques en temps réel sur le tableau de bord
  • Permettre l'export et l'import des données (JSON, CSV)

Objectifs techniques

  • Développer une SPA (Single Page Application) sans rechargement
  • Utiliser le localStorage pour la persistance des données
  • Implémenter une architecture JavaScript orientée objet (POO)
  • Assurer un design responsive adapté à tous les écrans
  • Respecter les normes d'accessibilité WCAG 2.1

4 Spécifications générales

Architecture de l'application

L'application sera structurée en deux pages principales au sein d'un fichier HTML unique :

  • Formulaire d'authentification avec identifiant et mot de passe
  • Validation des champs côté client
  • Message d'erreur personnalisé en cas d'échec
  • Design premium centré avec effet glassmorphism
  • Logo et nom du salon en vedette

Identifiants par défaut :

  • Identifiant : admin
  • Mot de passe : ndcolor2025
  • Header : Logo, navigation et bouton déconnexion
  • Statistiques : 5 cartes affichant les compteurs en temps réel
  • Barre de recherche : Filtrage instantané multicritères
  • Tableau de données : Liste complète des clients avec tri et pagination
  • Actions : Ajouter, modifier, supprimer, ajouter des points
  • Export/Import : JSON et CSV
  • Footer : Copyright et crédits

5 Fonctionnalités détaillées

5.1 Authentification

Système de connexion simple avec gestion de session :

  • Vérification des identifiants côté client
  • Stockage de l'état de connexion dans sessionStorage
  • Déconnexion automatique à la fermeture du navigateur
  • Protection de l'accès au dashboard sans authentification

5.2 Gestion des clients (CRUD)

CREATE : Ajout d'un nouveau client via un formulaire modal

Champs du formulaire client :

Champ Type Obligatoire Validation
Nom Texte Oui Min. 2 caractères
Prénom Texte Oui Min. 2 caractères
Téléphone Téléphone Oui Format français (10 chiffres)
Email Email Non Format email valide
Points Nombre Oui Entier ≥ 0, défaut : 0
Date création Date Auto Générée automatiquement

5.3 Système de cartes de fidélité

4 niveaux de cartes attribués automatiquement selon le nombre de points :

Bronze

0 - 99 points

Silver

100 - 299 points

Gold

300 - 599 points

VIP Platinum

600+ points

5.4 Recherche et filtrage

  • Recherche en temps réel (keyup event)
  • Filtrage multicritères : nom, prénom, email, téléphone
  • Insensible à la casse
  • Affichage des résultats instantané

5.5 Tri des colonnes

  • Tri ascendant/descendant sur chaque colonne
  • Indicateur visuel de la direction du tri
  • Tri alphabétique pour les textes
  • Tri numérique pour les points
  • Tri chronologique pour les dates

5.6 Pagination

  • Options d'affichage : 5, 10, 25, 50, 100 éléments par page
  • Navigation : première, précédente, numéros de page, suivante, dernière
  • Indicateur de position (ex: "Affichage 1-10 sur 45")
  • Accès direct à une page spécifique

5.7 Export/Import

Export
  • JSON : Export complet des données
  • CSV : Compatible Excel/LibreOffice
  • Nom de fichier avec date automatique
Import
  • Import de fichiers JSON uniquement
  • Choix : Fusionner ou Remplacer
  • Validation du format des données

5.8 Statistiques temps réel

Le tableau de bord affiche 5 cartes de statistiques :

  1. Total clients : Nombre total de clients enregistrés
  2. Cartes bronze : Nombre de clients avec carte Bronze
  3. Cartes silver : Nombre de clients avec carte Silver
  4. Cartes gold : Nombre de clients avec carte Gold
  5. Cartes VIP platinum : Nombre de clients VIP

8 Accessibilité (WCAG 2.1)

Objectif Lighthouse : L'application doit obtenir un score de 100% en accessibilité lors de l'audit Lighthouse de Chrome DevTools.

8.1 Structure sémantique HTML5

  • Utiliser les balises <header>, <main>, <footer>, <nav>, <article>, <section>
  • Hiérarchie correcte des titres (h1 → h2 → h3...)
  • Attributs role pour les régions importantes
  • Attribut lang="fr" sur la balise <html>

8.2 Navigation au clavier

  • Lien "Skip to content" en premier élément focusable
  • Ordre de tabulation logique (tabindex)
  • Indicateur de focus visible (:focus-visible)
  • Tous les éléments interactifs accessibles au clavier

8.3 ARIA labels

<!-- Exemples d'attributs ARIA requis --> <nav role="navigation" aria-label="Navigation principale"> <main id="main-content" role="main"> <button aria-label="Ajouter un nouveau client"> <input aria-label="Rechercher un client"> <table aria-describedby="tableDescription"> <div role="alert" aria-live="polite">

8.4 Formulaires accessibles

  • Labels associés à chaque champ (<label for="...">)
  • Messages d'erreur explicites
  • Attributs required et aria-required
  • Attributs autocomplete appropriés

8.5 Contrastes et couleurs

  • Ratio de contraste minimum 4.5:1 pour le texte normal
  • Ratio de contraste minimum 3:1 pour les grands textes
  • Information non transmise uniquement par la couleur

8.6 Préférences utilisateur

/* Respecter les préférences de réduction de mouvement */ @media (prefers-reduced-motion: reduce) { *, *::before, *::after { animation-duration: 0.01ms !important; animation-iteration-count: 1 !important; transition-duration: 0.01ms !important; } }
